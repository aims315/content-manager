タスク管理アプリ　引き継ぎ手順書

1. 渡すもの一覧

渡すもの内容・確認場所

2. 受け取った側のセットアップ手順

STEP 1. Node.js をインストール

https://nodejs.org にアクセスして LTS版 をダウンロード・インストール

STEP 2. Vercel CLI をインストール

ターミナル（Macはターミナル.app、Windowsはコマンドプロンプト）で：

npm install -g vercel

STEP 3. ZIPを展開してフォルダへ移動

cd task-management-app

STEP 4. パッケージをインストール

npm install

STEP 5. 環境変数ファイルを作成

フォルダ直下に .env.local というファイルを作り、以下を記入：

NEXT_PUBLIC_SUPABASE_URL=（SupabaseのProject URL）
NEXT_PUBLIC_SUPABASE_ANON_KEY=（SupabaseのAnon Key）
ADMIN_PATH_SECRET=qk9m3xtv
DISCORD_WEBHOOKS={"シンコール":"https://...","ALSOK":"https://...",...}

STEP 6. Vercelにデプロイ

vercel login    # Vercelアカウントでログイン（ブラウザが開く）
vercel --prod   # 本番環境にデプロイ

完了すると本番URLが表示されます。

3. Vercelアカウントごと引き継ぐ場合（簡単）

デプロイ不要で最も簡単な方法：

  1. Vercel（https://vercel.com）にログイン
  2. プロジェクト → Settings → Members → Invite
  3. 引き継ぎ先のメールアドレスを招待

Supabaseも同様：Supabase → Settings → Team → Invite member

6. DBを新規作成する場合のSQL

既存のSupabaseを引き継げばSQL不要。新しく作る場合のみ実行：

-- tasks テーブル
CREATE TABLE tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  description TEXT,
  assignee TEXT NOT NULL,
  status TEXT DEFAULT '未着手',
  due_date DATE,
  file_urls TEXT[] DEFAULT '{}',
  file_names TEXT[] DEFAULT '{}',
  client_slug TEXT,
  amount INTEGER,
  discord_channels TEXT[] DEFAULT '{}',
  modification_note TEXT,
  modification_files TEXT[] DEFAULT '{}',
  modified_by TEXT,
  modified_at TIMESTAMPTZ,
  response_url TEXT,
  response_note TEXT,
  responded_at TIMESTAMPTZ,
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- task_revisions テーブル
CREATE TABLE task_revisions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID REFERENCES tasks(id) ON DELETE CASCADE,
  note TEXT NOT NULL,
  created_by TEXT NOT NULL,
  file_urls TEXT[] DEFAULT '{}',
  file_names TEXT[] DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- client_settings テーブル
CREATE TABLE client_settings (
  slug TEXT PRIMARY KEY,
  discord_channel TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- RLS（全員アクセス許可）
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
ALTER TABLE task_revisions ENABLE ROW LEVEL SECURITY;
ALTER TABLE client_settings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "allow all" ON tasks FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all" ON task_revisions FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all" ON client_settings FOR ALL USING (true) WITH CHECK (true);

-- Storage（task-filesバケットをPublicで作成後）
CREATE POLICY "allow upload" ON storage.objects
  FOR INSERT TO anon WITH CHECK (bucket_id = 'task-files');
CREATE POLICY "allow read" ON storage.objects
  FOR SELECT TO anon USING (bucket_id = 'task-files');
