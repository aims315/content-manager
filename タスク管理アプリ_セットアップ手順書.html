1. システム概要・使い方

何ができるシステムか

デザイン・動画などの制作案件を管理するためのWebアプリです。管理者とクライアント（お客様）がそれぞれ専用の画面を持ちます。

画面誰が使うできること
管理画面制作会社側タスク一覧・ステータス管理・カレンダー（初校/最終切り替え）・カテゴリ絞り込み・初校提出・修正対応
タスク送信フォーム制作会社側新規タスク登録・修正指示の送信
クライアントポータルお客様新規依頼・修正指示・進捗確認・ファイル確認

ステータスの流れ

タスクは以下の順序でステータスが変わります：

未着手 → 制作要項待ち → 制作要項受領 → 進行中 → 初校提出 → 修正 → 修正対応完了 → 完了

使用しているサービス

サービス名役割費用
VercelWebサイトの公開サーバー無料プランあり
Supabaseデータベース・ファイル保存無料プランあり
Discord通知（Webhook）無料
Resendメール通知送信月3,000通まで無料

4. Supabase（データベース）のセットアップ

4-1. 新しいプロジェクトを作成

1

Supabaseにログイン → 「New Project」をクリック

2

プロジェクト名（例：task-management）、データベースパスワード（忘れずメモ）を入力して「Create new project」

3

2〜3分待つとプロジェクトが作成されます

4-2. テーブル（データの入れ物）を作成

左メニューの「SQL Editor」→「New query」をクリックして、以下のSQLをコピー＆ペーストして「Run」ボタンを押してください。

⚠️ 下のSQLは必ず一度に全部コピーして実行してください。
-- =============================================
-- タスク管理アプリ データベース初期設定SQL（最新版）
-- =============================================

-- 1. タスクテーブル
CREATE TABLE IF NOT EXISTS tasks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  description TEXT,
  assignee TEXT NOT NULL DEFAULT '',
  status TEXT DEFAULT '未着手'
    CONSTRAINT tasks_status_check
    CHECK (status IN ('未着手', '制作要項待ち', '制作要項受領', '進行中', '初校提出', '修正', '修正対応完了', '完了')),
  due_date DATE,
  file_urls TEXT[] DEFAULT '{}',
  file_names TEXT[] DEFAULT '{}',
  client_slug TEXT,
  amount INTEGER,
  staff TEXT,
  discord_channels TEXT[] DEFAULT '{}',
  modification_note TEXT,
  modification_files TEXT[] DEFAULT '{}',
  modified_by TEXT,
  modified_at TIMESTAMPTZ,
  response_url TEXT,
  response_note TEXT,
  responded_at TIMESTAMPTZ,
  response_file_urls TEXT[] DEFAULT '{}',
  response_file_names TEXT[] DEFAULT '{}',
  draft_url TEXT,
  draft_note TEXT,
  draft_submitted_at TIMESTAMPTZ,
  draft_file_urls TEXT[] DEFAULT '{}',
  draft_file_names TEXT[] DEFAULT '{}',
  draft_due_date DATE,
  completed_at TIMESTAMPTZ,
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. 修正指示テーブル
CREATE TABLE IF NOT EXISTS task_revisions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID REFERENCES tasks(id) ON DELETE CASCADE,
  note TEXT NOT NULL,
  created_by TEXT NOT NULL DEFAULT '',
  file_urls TEXT[] DEFAULT '{}',
  file_names TEXT[] DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 3. クライアント設定テーブル（Discord・メール設定）
CREATE TABLE IF NOT EXISTS client_settings (
  slug TEXT PRIMARY KEY,
  discord_channel TEXT,
  notification_email TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 4. セキュリティ設定（RLS：全員アクセス許可）
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
ALTER TABLE task_revisions ENABLE ROW LEVEL SECURITY;
ALTER TABLE client_settings ENABLE ROW LEVEL SECURITY;

CREATE POLICY "allow all" ON tasks FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all" ON task_revisions FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all" ON client_settings FOR ALL USING (true) WITH CHECK (true);

-- 5. リアルタイム更新を有効化
ALTER TABLE tasks REPLICA IDENTITY FULL;
ALTER TABLE task_revisions REPLICA IDENTITY FULL;
ALTER TABLE client_settings REPLICA IDENTITY FULL;
✅ 「Success. No rows returned」と表示されれば成功です！

4-3. ファイル保存用ストレージの設定

1

左メニューの「Storage」→「New bucket」をクリック

2

Bucket name に task-files と入力、「Public bucket」をONにして「Save」

⚠️ 必ず「Public bucket」をONにしてください。ONにしないとファイルが表示されません。
3

SQL Editorで以下を実行してアップロード権限を設定：

-- ファイルアップロードを許可
CREATE POLICY "allow upload" ON storage.objects
  FOR INSERT TO anon WITH CHECK (bucket_id = 'task-files');

CREATE POLICY "allow read" ON storage.objects
  FOR SELECT TO anon USING (bucket_id = 'task-files');

CREATE POLICY "allow delete" ON storage.objects
  FOR DELETE TO anon USING (bucket_id = 'task-files');

5. Vercel（公開サーバー）のセットアップ

5-1. ソースコードの準備

1

受け取ったZIPファイルを解凍する

2

解凍したフォルダの中に .env.local という名前のファイルを新規作成して、以下を記入：

NEXT_PUBLIC_SUPABASE_URL=（SupabaseのProject URL）

6. Discord通知の設定

Webhook URLの取得方法

1

Discordで通知を受け取りたいチャンネルを右クリック → 「チャンネルの編集」

2

「連携サービス」→「ウェブフック」→「新しいウェブフック」

3

「ウェブフックURLをコピー」でURLをコピー

4

複数チャンネルある場合は以下の形式でまとめてVercelに設定：

{
  "チャンネル名A": "https://discord.com/api/webhooks/...",
  "チャンネル名B": "https://discord.com/api/webhooks/...",
  "default": "https://discord.com/api/webhooks/..."

💡 default はクライアントコードとの紐付けがない場合に使われるチャンネルです。必ず設定してください。

10. トラブルシューティング

「This page couldn't load」と表示される

JavaScriptのエラーが起きています。以下を確認してください：

  • Vercelの環境変数がすべて正しく設定されているか
  • vercel --prod で最新版をデプロイしたか

ファイルがアップロードできない・表示されない

Supabaseのストレージ設定を確認：

  • Storage → task-filesバケットが「PUBLIC」になっているか
  • Storage Policyが設定されているか（INSERT / SELECT / DELETEの3つ）

Discord通知が届かない

  • DISCORD_WEBHOOKSの値がJSON形式で正しく設定されているか確認
  • Webhook URLが有効かDiscordで確認
  • default キーが設定されているか確認

ステータスの変更でエラーが出る

DBのステータス制約が古い可能性があります。SQL Editorで実行：

UPDATE tasks SET status = '未着手'
  WHERE status NOT IN ('未着手','制作要項待ち','制作要項受領','進行中','初校提出','修正','修正対応完了','完了');
ALTER TABLE tasks DROP CONSTRAINT IF EXISTS tasks_status_check;
ALTER TABLE tasks ADD CONSTRAINT tasks_status_check
  CHECK (status IN ('未着手', '制作要項待ち', '制作要項受領', '進行中', '初校提出', '修正', '修正対応完了', '完了'));

既存のDBにカラムを追加する（アップデート時）

新バージョンで追加されたカラムが不足している場合はSQL Editorで実行：

-- 担当者・ゴミ箱・初校ファイル・納品ファイルカラムを追加
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS staff TEXT;
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ;
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS draft_file_urls TEXT[] DEFAULT '{}';
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS draft_file_names TEXT[] DEFAULT '{}';
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS response_file_urls TEXT[] DEFAULT '{}';
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS response_file_names TEXT[] DEFAULT '{}';
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS draft_due_date DATE;
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS amount INTEGER;
ALTER TABLE client_settings ADD COLUMN IF NOT EXISTS notification_email TEXT;

-- 締切リマインダー設定テーブル（Vercel Cron用）
CREATE TABLE IF NOT EXISTS app_settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT now()
);
ALTER TABLE app_settings ENABLE ROW LEVEL SECURITY;
-- service_role でのみアクセス（RLS で anon を拒否）
CREATE POLICY "service only" ON app_settings USING (false);
INSERT INTO app_settings (key, value) VALUES ('due_reminder_days', '3') ON CONFLICT (key) DO NOTHING;

メール通知が届かない

  • RESEND_API_KEYが正しく設定されているか確認
  • クライアントポータルの「進捗確認」タブでメールアドレスが保存されているか確認
  • 迷惑メールフォルダも確認

11. 別の会社・別の管理画面として新たに作る手順

このアプリを別の会社・別のプロジェクト向けに新たにセットアップしてあげる場合の手順です。コード（ZIPファイル）は使い回しできます。作るのは「Supabaseのプロジェクト」と「Vercelのプロジェクト」だけです。

💡 ポイント：コードは同じもの・変わるのは「環境変数」だけです。
新しい Supabase プロジェクト ＋ 新しい Vercel プロジェクト ＝ 完全に独立した別の管理画面ができます。

自分のPCに必要なもの（事前確認）

  • ✅ /Users/あなた/Downloads/task-management-app.zip（アプリのZIP）
  • ✅ Supabase のアカウント（既存のもので別プロジェクトを追加できる）
  • ✅ Vercel のアカウント（既存のもので別プロジェクトを追加できる）
  • ✅ 相手先の Discord Webhook URL（相手に作ってもらうか、自分で作る）

全体の流れ（30〜60分）

①
新しい Supabase プロジェクトを作る
②
SQL を実行してテーブルを作る
③
Storage のバケットを作る
④
新しい Vercel プロジェクトを作る（同じZIPを使う）
⑤
環境変数を設定する
⑥
デプロイして動作確認・URLを相手に渡す

① 新しい Supabase プロジェクトを作る

1

supabase.com にログインして「New project」をクリック

  • Name：相手先の会社名など（例：yamada-tasks）
  • Database Password：自動生成されたものをどこかにメモ
  • Region：Northeast Asia（Tokyo）を選択

作成完了まで1〜2分待ちます。

2

プロジェクトが作成されたら Project Settings → API を開いて以下の値をメモする：

  • Project URL（例：https://xxxxxxxx.supabase.co）
  • anon public キー（長い文字列）
  • service_role キー（長い文字列・秘密）
⚠️ service_role キーは絶対に相手に渡さないでください。Vercelの環境変数にだけ設定します。

② SQLを実行してテーブルを作る

新しいプロジェクトの SQL Editor を開いて、以下のSQLを貼り付けて実行します。

-- タスクテーブル

  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  description TEXT,
  assignee TEXT NOT NULL DEFAULT '',
  due_date DATE,
  status TEXT NOT NULL DEFAULT '未着手'
    CONSTRAINT tasks_status_check
    CHECK (status IN ('未着手','進行中','初校提出','修正','修正対応完了','完了')),
  file_urls TEXT[] DEFAULT '{}',
  file_names TEXT[] DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ,
  modification_note TEXT,
  modification_files TEXT[] DEFAULT '{}',
  modified_by TEXT,
  modified_at TIMESTAMPTZ,
  discord_channels TEXT[] DEFAULT '{}',
  response_url TEXT,
  response_note TEXT,
  responded_at TIMESTAMPTZ,
  draft_url TEXT,
  draft_note TEXT,
  draft_submitted_at TIMESTAMPTZ,
  draft_due_date DATE,
  client_slug TEXT,
  amount INTEGER
);

-- 修正指示テーブル
CREATE TABLE IF NOT EXISTS task_revisions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id UUID NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
  note TEXT NOT NULL,
  created_by TEXT NOT NULL DEFAULT '',
  file_urls TEXT[] DEFAULT '{}',
  file_names TEXT[] DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- クライアント設定テーブル
CREATE TABLE IF NOT EXISTS client_settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  slug TEXT UNIQUE NOT NULL,
  discord_channel TEXT,
  notification_email TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Realtimeを有効化（管理画面とクライアントポータルの即時反映に必要）
ALTER TABLE tasks REPLICA IDENTITY FULL;
ALTER TABLE task_revisions REPLICA IDENTITY FULL;
ALTER TABLE client_settings REPLICA IDENTITY FULL;

「Success. No rows returned.」と表示されれば完了です。

③ Storage のバケットを作る

1

Supabase の左メニュー Storage → New bucket をクリック

  • Name：task-files（この名前で固定）
  • Public bucket：✅ オンにする
2

バケット作成後、SQL Editor で以下を実行してアップロードを許可する：

-- 誰でもファイルをアップロードできるポリシー
CREATE POLICY "allow upload" ON storage.objects
  FOR INSERT TO anon WITH CHECK (bucket_id = 'task-files');

-- 誰でもファイルを読める（公開）ポリシー
CREATE POLICY "allow read" ON storage.objects
  FOR SELECT TO anon USING (bucket_id = 'task-files');

④ 新しい Vercel プロジェクトを作る

1

vercel.com にログインして Add New → Project をクリック

2

「Import Git Repository」ではなく「Deploy without a Git repository（ZIP）」を選ぶ

または、GitHubリポジトリを持っている場合は同じリポジトリを再度インポートしてもOK（別プロジェクトとして作成される）

⚠️ ZIPからの直接インポート：Vercelのダッシュボードで「Import」→「Upload」→ ZIPを選択
3

プロジェクト名を決める（例：yamada-task-management）

Framework Preset は Next.js を選択

まだ「Deploy」は押さない → 先に環境変数を設定する

⑤ 環境変数を設定する

「Environment Variables」のセクションに以下を1つずつ追加します。

変数名値備考
NEXT_PUBLIC_SUPABASE_URL 新しいプロジェクトのURL Supabase → Settings → API → Project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY 新しい anon キー Supabase → Settings → API → anon public
SUPABASE_SERVICE_ROLE_KEY 新しい service_role キー Supabase → Settings → API → service_role
ADMIN_PATH_SECRET 新しい管理パス（自分で決める） 例：abc12345　※元の案件と別の値を推奨
DISCORD_WEBHOOKS 相手先のDiscord Webhook JSON 例：{"チャンネル名":"https://discord.com/..."}
RESEND_API_KEY 自分のResend APIキー（使い回し可） メール通知が不要なら設定しなくてもOK
💡 ADMIN_PATH_SECRET について：これが管理画面のURLになります。
例えば yamada2024 に設定すると、管理画面URLは
https://yamada-task-management.vercel.app/p/yamada2024 になります。
相手に渡すURLと、クライアント用URLを整理してメモしておきましょう。

⑥ デプロイして相手に渡す

1

環境変数の設定後、「Deploy」ボタンを押す。2〜3分でデプロイ完了。

2

完了後、以下のURLを相手に伝える：

渡す相手URL
管理側（あなたたち） https://[プロジェクト名].vercel.app/p/[ADMIN_PATH_SECRET]
クライアント側 https://[プロジェクト名].vercel.app/submit/[クライアントコード]
Discord通知設定 https://[プロジェクト名].vercel.app/p/[ADMIN_PATH_SECRET]/settings
3

管理画面にアクセスして動作確認：

  • テストタスクを1件作成してみる
  • クライアントポータルにアクセスしてタスクが見えるか確認
  • Discord通知設定からチャンネルを紐付ける
✅ これで完全に独立した別の管理画面が完成です。
元の案件のデータとは完全に分離されています。

よくある質問

Q. 何社分でも作れる？

はい。Supabase（無料プランで最大2プロジェクト、有料で無制限）とVercel（無料プランで無制限プロジェクト）の制限範囲内であれば何社分でも作れます。

Q. Resend（メール通知）は使い回せる？

はい。同じ API キーを複数のVercelプロジェクトで使えます。ただし無料プランは月100通まで。複数社で使う場合は通数に注意してください。

Q. 独自ドメインを使いたい

Vercel の各プロジェクトの Settings → Domains から独自ドメインを追加できます。例：tasks.yamada-company.com

Q. 相手に GitHub や Vercel のアカウントを作ってもらう必要はある？

いいえ。自分のアカウントで全部管理する場合は不要です。相手には管理画面のURLとパスワード（ADMIN_PATH_SECRET）だけ渡せばOKです。相手側で自分たちが管理したい場合は別途アカウントを作って引き継ぎます。

12. GitHub連携でターミナル不要の自動デプロイ

現在はターミナルで vercel --prod を実行してデプロイしています。GitHubと連携すると、ファイルを保存してGitHubにアップするだけで自動的にVercelにデプロイされます。ターミナルは不要になります。

💡 メリット：コードの変更履歴が残る・間違えても前のバージョンに戻せる・デプロイが自動化される

① GitHubアカウントを作る（持っている場合はスキップ）

1

github.com にアクセスして「Sign up」でアカウント作成

  • メールアドレス・パスワード・ユーザー名を設定するだけ
  • 無料プランで十分です

② GitHub Desktop をインストール（GUI操作ツール）

ターミナル不要でGitHubを使えるアプリです。

1

desktop.github.com からダウンロード・インストール

起動後、GitHubアカウントでサインイン

③ アプリをGitHubに登録（リポジトリ作成）

1

GitHub Desktop を開いて 「Add an Existing Repository from your Hard Drive...」 をクリック

→ /Users/あなた/Downloads/task-management-app フォルダを選択

2

「create a repository」リンクが出たらクリック → Initialize Repository ボタンを押す

3

右上の 「Publish repository」 ボタンを押す

  • Name：task-management-app（そのままでOK）
  • Keep this code private：✅ チェックを入れる（他人に見られないように）

「Publish Repository」ボタンを押して完了

④ VercelをGitHubリポジトリに連携させる

1

vercel.com → 該当プロジェクト → Settings → Git を開く

2

「Connect Git Repository」 をクリック → GitHub を選択 → 先ほど作った task-management-app リポジトリを選択

3

接続完了。これ以降は自動デプロイが有効になります。

✅ これ以降の更新は以下の手順だけ：

① ファイルを編集・保存
② GitHub Desktop を開く
③ 左下の「Summary」欄に変更内容を一言入力（例：「担当者機能を追加」）
④ 青い 「Commit to main」 ボタンを押す
⑤ 上の 「Push origin」 ボタンを押す
⑥ 1〜2分でVercelに自動デプロイ完了 🎉

間違えた時に前のバージョンに戻す方法

1

GitHub Desktop の 「History」タブ を開く → 過去のコミット一覧が表示される

2

戻したいコミットを右クリック → 「Revert Changes in Commit」 を選択

→ Push origin すれば古いバージョンに戻ってデプロイされる

⚠️ 注意：環境変数（APIキーなど）はVercelで管理されているので、GitHubには保存されません。新しいプロジェクトを作る時は環境変数の再設定が必要です。

13. 締切リマインダー通知（Vercel Cron）

指定日数後に締切が来るのに未完了のタスクを、毎朝 9:00 JST（0:00 UTC） に自動でDiscordへ通知します。

① Supabase でテーブル作成

SupabaseのSQL Editorで以下を実行してください（初回のみ）：

CREATE TABLE IF NOT EXISTS app_settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT now()
);
ALTER TABLE app_settings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "service only" ON app_settings USING (false);
INSERT INTO app_settings (key, value)
  VALUES ('due_reminder_days', '3')
  ON CONFLICT (key) DO NOTHING;

② Vercel 環境変数の確認

SUPABASE_SERVICE_ROLE_KEY が Vercel に設定されていること（Supabase → Project Settings → API → service_role key）

⚠️ service_role キーは絶対に公開しないでください。Vercelの環境変数にのみ設定します。

③ デプロイするだけで自動有効化

vercel.json にCron設定が含まれているため、デプロイするだけで Vercel Cron が自動的に有効になります。

📋 動作確認はVercelダッシュボード → Cron Jobs タブで確認できます。手動実行ボタンもあります。

④ 通知日数の変更方法

管理画面のフィルタバーに 「締切○日前に未完了通知」 という設定バッジが表示されます。「変更」をクリックして日数を入力→「保存」すると Supabase に保存され、次回のCronから新しい日数で通知されます。
