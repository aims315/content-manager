# 他社展開ガイド（コンテンツ制作管理アプリ）

別の会社で同じアプリを使うための手順です。
**データは会社ごとに完全に分かれます**（新しいSupabaseを作るため）。

---

## 必要なアカウント（すべて企業アカウントで）

1. **GitHub** … コードの置き場
2. **Vercel** … サイトの公開（GitHubと連携）
3. **Supabase** … データベース（新規作成）

---

## STEP 1：GitHub にコードを置く

おすすめは「新規リポジトリにコピー」です（元と切り離せて安全）。

1. 企業のGitHubアカウントで新しいリポジトリを作成（例：`content-manager`、Privateでよい）
2. このコードをそのリポジトリにpushする
   （※この作業は私（Claude）が手伝えます。リポジトリURLを教えてください）

---

## STEP 2：Supabase で新しいデータベースを作る

1. https://supabase.com で企業アカウントにログイン
2. 「New project」で新規プロジェクト作成
   - 名前：会社名など分かりやすい名前
   - Database Password：控えておく
   - Region：`Northeast Asia (Tokyo)` 推奨
3. 作成後、左メニュー **SQL Editor** を開く
4. リポジトリ内の **`supabase_migration.sql`** の中身を全部コピー＆ペースト
5. **Run** を押す（テーブル・権限・ストレージが一括作成される）
6. 左メニュー **Project Settings → API** を開き、以下2つをメモ：
   - **Project URL**（`https://xxxx.supabase.co`）
   - **anon public** キー
   - **service_role** キー（※秘密。他人に見せない）

---

## STEP 3：Vercel で公開

1. https://vercel.com で企業アカウントにログイン
2. 「Add New → Project」→ STEP1のGitHubリポジトリを選んでImport
3. **Environment Variables（環境変数）** に下記を入力（下の表参照）
4. **Deploy** を押す
5. 数分でURLが発行される（`xxxx.vercel.app`）

---

## 環境変数一覧（Vercelに入れるもの）

| 変数名 | 必須 | 内容 |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | ✅ | Supabaseの Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ | Supabaseの anon public キー |
| `SUPABASE_SERVICE_ROLE_KEY` | ✅ | Supabaseの service_role キー |
| `ADMIN_PATH_SECRET` | ✅ | 新規作成ページのURL用の合言葉（好きな文字列、例：`mysecret2026`） |
| `CHATWORK_API_TOKEN` | 任意 | Chatwork通知を使う場合のAPIトークン |
| `CHATWORK_ROOM_ID` | 任意 | 通知先のChatworkルームID |
| `DISCORD_WEBHOOK_URL` | 任意 | Discord通知を使う場合 |
| `RESEND_API_KEY` | 任意 | メール通知を使う場合 |
| `CRON_SECRET` | 任意 | 締切リマインダー（定期実行）を使う場合の合言葉 |

> **最低限これだけでOK**：上の「✅」が付いた4つを入れれば動きます。
> 通知系（Chatwork等）は後からでも追加できます。

---

## STEP 4：初期設定（公開後）

1. 発行されたURLにアクセス
2. 右上の設定ボタン（⚙️）から「役割」「ステータス」をその会社用に設定
3. 「新規プロジェクト」ボタン or CSV取込でデータを入れる

---

## 困ったときの確認ポイント

- **画面が真っ白／エラー** → 環境変数（特にSupabase 3つ）が正しいか確認
- **データが保存されない** → SupabaseでSQLがちゃんと実行されたか（テーブルが出来ているか）確認
- **「新規プロジェクト」ページが開けない** → URLの `ADMIN_PATH_SECRET` 部分が環境変数と一致しているか確認
